# Contributing to The Programming Labyrinth

Thank you for your interest in contributing!

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Open the project in Godot 4.6+.

## Code Style

- Follow the existing GDScript style (snake_case for variables and functions).
- Add docstrings to new public functions.
- Keep scripts focused and single-responsibility.

## Running Tests

```bash
godot --headless --script res://tests/test_all.gd
```

All tests must pass before submitting a pull request.

## Pull Requests

- Describe what your PR changes and why.
- Reference any related issues.
- Ensure all tests pass.
